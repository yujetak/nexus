# AI Agent Development & Coding Conventions

이 파일은 프로젝트의 코드 품질과 일관성을 유지하기 위한 지침입니다. 
AI 에이전트는 모든 코드 생성, 리팩토링 및 문서화 작업 시 아래 규칙을 엄격히 준수해야 합니다.

## 1. Core Architecture Rules (아키텍처 규칙)
- **Role Isolation**: 
  - Spring Boot: 비즈니스 로직, 인증(Auth), 메인 DB CRUD 담당.
  - FastAPI: 데이터 분석, AI 로직, 비동기 작업 담당.
  - Next.js: UI/UX, 클라이언트 상태 관리, 서버 사이드 렌더링 담당.
- **Communication**: 서버 간 통신은 반드시 JSON 기반 REST API를 사용하며, Spring Boot에서 FastAPI 호출 시 공통 모듈(`ExternalApiService`)을 최우선으로 활용할 것.

## 2. Naming Conventions (명칭 규칙)
- **Variables & Functions**: `camelCase` (예: `getUserInfo`, `stockPrice`)
- **Classes & Components**: `PascalCase` (예: `StockController`, `UserCard`)
- **Database Tables/Columns**: `snake_case` (예: `user_account`, `created_at`)
- **Constants**: `SCREAMING_SNAKE_CASE` (예: `MAX_RETRY_COUNT`)

## 3. Programming Language Specifics (언어별 규칙)

### [Java / Spring Boot]
- **Annotations**: `@Getter`, `@Setter` 대신 필요한 경우에만 최소한으로 사용하고, 생성자 주입은 `@RequiredArgsConstructor`를 활용할 것.
- **Response**: 모든 컨트롤러는 `ResponseEntity<T>`를 반환하며, 공통 응답 포맷(Success/Error)을 유지할 것.
- **Documentation**: 신규 API 생성 시 반드시 Swagger(`@Operation`, `@Tag`) 주석을 작성할 것.

### [Python / FastAPI]
- **Typing**: 모든 함수 인자와 반환값에 'Type Hinting'을 적용할 것.
- **Validation**: 요청/응답 모델은 `Pydantic`을 사용하여 정의할 것.
- **Asynchronous**: 모든 경로는 `async def`를 기본으로 작성할 것.
- **Environment**: 파이썬 코드를 실행하거나 패키지를 설치할 때는 반드시 가상환경(`.venv`)을 생성하고 활성화한 상태에서 진행할 것.

### [TypeScript / Next.js]
- **Components**: Arrow Function 기반의 Functional Component를 사용할 것.
- **Props**: Interface를 정의하여 타입 안정성을 확보할 것.
- **Styling**: Tailwind CSS 클래스 조합을 우선하되, 가독성을 위해 긴 클래스는 줄바꿈을 활용할 것.

## 4. Error Handling & Logging (에러 및 로그)
- 모든 에러 응답은 사용자 친화적인 메시지와 함께 내부 추적을 위한 에러 코드를 포함할 것.
- `try-catch` 블록 사용 시 반드시 로그(`log.error` 또는 `console.error`)를 남길 것.
- 하드코딩된 에러 메시지 대신 공통 에러 메시지 상수를 활용할 것.

## 5. Clean Code Constraints (클린 코드 제약)
- 함수/메서드 하나는 가급적 하나의 기능만 수행하도록 작게 유지할 것 (최대 20라인 권장).
- 중복되는 로직은 반드시 공통 유틸리티 클래스나 커스텀 훅으로 분리할 것.
- 주석은 '어떻게(How)'보다 '왜(Why)'를 설명하는 데 집중할 것.
- 모든 아티팩트와 설명은 반드시 한국말로 작성하고, 프로젝트 관련 의사소통은 대표님에게 공손히 대할 것.

## 6. API & Network Conventions (API 및 네트워크 규정)

AI 에이전트는 API 설계 시 다음의 규정을 엄격히 준수하여 일관성을 유지해야 합니다.

### 📍 API Path & Routing
- **기본 Prefix**: 모든 API 주소는 `/api/v1/`으로 시작한다.
- **서버별 경로 분리**:
    - **FastAPI (AI 전용)**: `/api/v1/ai/**` (예: `/api/v1/ai/analyze`)
    - **Spring Boot (비즈니스)**: `/api/v1/auth/**`, `/api/v1/users/**` 등

### 🌐 Base URL Settings
- **Spring Boot**: `http://localhost:8080` (배포 시 환경변수 처리)
- **FastAPI**: `http://localhost:8000` (배포 시 환경변수 처리)

### 📥 HTTP Methods Use Case
- **GET**: 리소스 조회
- **POST**: 리소스 생성
- **PUT**: 리소스 전체 수정
- **PATCH**: 리소스 일부 수정
- **DELETE**: 리소스 삭제

### 🔐 Authentication (인증)
- **방식**: Bearer Token (Spring JWT 기반)
- **Header**: `Authorization: Bearer {access_token}`

### 🚀 Common Error Codes (공통 에러 코드)
에러 반환 시 다음의 코드를 상황에 맞게 사용합니다.
- **400 BAD_REQUEST**: 잘못된 요청 파라미터
- **401 UNAUTHORIZED**: 인증 토큰 누락 또는 만료
- **403 FORBIDDEN**: 해당 자원에 대한 권한 없음
- **404 NOT_FOUND**: 리소스를 찾을 수 없음
- **500 INTERNAL_SERVER_ERROR**: 서버 내부 오류 (AI 엔진 오류 포함)
